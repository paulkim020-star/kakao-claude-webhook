# CLAUDE.md — 카카오 Claude Webhook 프로젝트

## 프로젝트 개요

카카오 i 오픈빌더 챗봇과 Claude API를 연결하는 FastAPI 기반 웹훅 서버.

- **진입점**: `main.py`
- **엔드포인트**: `POST /kakao/webhook`, `GET /`
- **주요 의존성**: `fastapi`, `anthropic`, `uvicorn`

## 아키텍처

```
카카오톡 사용자 → 카카오 i 오픈빌더 → POST /kakao/webhook → Claude API → 카카오 스킬 응답(JSON)
```

- 사용자 대화 기록은 인메모리(`_conversation_store`)에 최대 `MAX_HISTORY_TURNS=6` 턴 보관
- 운영 환경에서는 Redis 등 외부 저장소로 교체 필요 (서버 재시작 시 초기화됨)
- 카카오 스킬 서버는 **5초 이내** 응답해야 타임아웃 없음

## 환경 변수

| 변수 | 필수 | 설명 |
|------|------|------|
| `ANTHROPIC_API_KEY` | 필수 | Anthropic 콘솔에서 발급 |
| `CLAUDE_MODEL` | 선택 | 기본값: `claude-sonnet-4-6` |
| `SYSTEM_PROMPT` | 선택 | 챗봇 페르소나 지정 |

## 로컬 실행

```bash
pip install -r requirements.txt
# .env 파일에 ANTHROPIC_API_KEY=sk-ant-... 작성
uvicorn main:app --host 0.0.0.0 --port 8000
```

## 코딩 가이드라인 (Karpathy 원칙 기반)

### 1. 코딩 전 먼저 생각하기

- 작업 전에 가정을 명시적으로 드러내고, 불명확한 부분은 질문한다
- 여러 해석이 가능한 요청은 침묵 속에 하나를 선택하지 말고 트레이드오프를 제시한다
- 혼란을 숨기지 않는다

### 2. 단순함 우선

- 문제를 해결하는 **최소한의 코드**만 작성한다
- 요청하지 않은 기능, 일회성 추상화, 발생 가능성 낮은 시나리오의 에러 핸들링은 추가하지 않는다
- 비슷한 코드 3줄이 성급한 추상화보다 낫다

### 3. 외과적 변경

- 반드시 건드려야 하는 부분만 수정한다
- 기존 코드 스타일을 유지한다 (포매팅, 네이밍 컨벤션 등)
- 관련 없는 코드를 리팩토링하거나 정리하지 않는다
- 자신이 만든 변경으로 인해 불필요해진 코드만 제거한다

### 4. 목표 기반 실행

- 작업을 시작하기 전에 **검증 가능한 완료 기준**을 정의한다
- 완료 기준이 충족될 때까지 반복한다
- 실제로 확인하지 않고 "완료"라고 선언하지 않는다

## 이 프로젝트에서 특히 주의할 점

- **카카오 스킬 응답 포맷**: 반드시 `_kakao_simple_text_response()` 포맷을 유지해야 오픈빌더가 파싱 가능
- **타임아웃**: Claude API 응답이 5초를 초과할 경우 콜백(useCallback) 방식으로 확장 필요
- **대화 기록**: `_conversation_store`는 프로세스 재시작 시 초기화됨 — 운영 환경 전환 시 Redis 도입 검토
- **API 키 노출 금지**: `.env`는 절대 커밋하지 않음
